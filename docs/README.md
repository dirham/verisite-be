# Docs Index

Use this folder as the working source for implementation, not as passive reference material.

## Reading Order

1. `development-workflow.md`
2. `implementation-plan.md`
3. `architecture.md`
4. `spec-gap-notes.md`

## Purpose Of Each File

- `development-workflow.md`: how to branch, plan, slice, verify, and update docs while pairing with Codex
- `implementation-plan.md`: the main execution checklist and current backlog
- `architecture.md`: the intended project structure and implementation boundaries
- `spec-gap-notes.md`: contract gaps that may block or shape implementation choices

## Rules

- Start new feature work from an up-to-date `master`.
- Use checklist items for work tracking.
- Use `[]` when a planned list is intentionally empty.
- Mark completed work with `[x]` only when the implementation really exists.
- Update docs in the same change as code whenever behavior or workflow changes.
